# Red Pitaya Project Configuration Summary: LED Blinky

## 1. Hardware & Software Environment
*   **Board:** Red Pitaya StemLab 125-14 v1.1[cite: 1]
*   **SoC (FPGA/CPU):** Xilinx Zynq-7000 (XC7Z010-1CLG400C)[cite: 1]
*   **Project Root:** `C:\Xilinx\RedPitaya`[cite: 1]
*   **FPGA Tool:** Vivado 2020.1 (Windows)[cite: 1]
*   **Software Requirement:** None (Pure Hardware Logic).
*   **Project Target:** Internal FPGA counter to drive physical LEDs.

## 2. Repository & Project Selection

*   **Repository Branch/Tag:** `v2020.1`[cite: 1]

---

## 3. Hardware Build Settings (Vivado Windows)
*   **Project Path:** `C:\Xilinx\RedPitaya\prj\v0.94`[cite: 1]
*   **Logic Implementation:** 
    *   **Clock:** Use the existing 125 MHz system clock (`clk_fpga_0`).[cite: 1]
    *   **Counter:** Create a 27-bit synchronous counter in Verilog/VHDL.
    *   **Mapping:** Assign the most significant bit (MSB) of the counter to `led_o[0]`.
*   **Constraint File:** Edit `red_pitaya_v0.94.xdc` to ensure `led_o` pins are correctly mapped to the Zynq package pins.
*   **Export Format:** `.bit` (Bitstream only).



---

## 4. Simplified Workflow (Hardware Only)
1.  **Initialize Project:** Open Vivado Tcl Shell and run:
    `vivado -mode batch -source make_project.tcl -tclargs MODEL=Z10`[cite: 1]
2.  **Modify RTL:** Open the generated project in the Vivado GUI. Find the top-level module and bypass the AXI registers. Create a simple divider:
    ```verilog
    reg [26:0] counter;
    always @(posedge clk_i) counter <= counter + 1'b1;
    assign led_o = counter[26:19]; // Blinks multiple LEDs at different rates
    ```
3.  **Generate Bitstream:** Run **Synthesis**, **Implementation**, and **Generate Bitstream** within Vivado.
4.  **Locate File:** Find the `red_pitaya_top.bit` in the implementation folders.

---

## 5. Deployment (No SDK Required)
1.  **Transfer:** Use `scp` (via PowerShell or WSL) to send the `.bit` file to the Red Pitaya:
    `scp red_pitaya_top.bit root@rp-XXXXXX.local:/root/`
2.  **Execute:** Log in via SSH and load the bitstream directly into the FPGA:
    `cat red_pitaya_top.bit > /dev/xdevcfg`
3.  **Verify:** The onboard yellow LEDs should immediately begin blinking.

---

## 6. Key Prerequisites
*   **No Linux App Needed:** Since the logic is self-running in the PL (Programmable Logic), you don't need to write a C program or use `mmap()` for this test.
*   **Clocking:** Ensure the Zynq PS (Processor System) is active, as it usually provides the `fclk0` used by the FPGA logic.[cite: 1]
